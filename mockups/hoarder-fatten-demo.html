The Actual Godot Shader
hoarder_fatten.gdshader — canvas_item, one material reused for all enemies
shader_type canvas_item;

// Hoarder "fatten": belly-bulge UV distortion.
// Sprite frames need transparent padding (~15%) or the bulge clips at the texture edge.
uniform float fatness : hint_range(0.0, 1.0) = 0.83;
uniform vec2  belly_center = vec2(0.5, 0.58);  // tweak per body shape if needed
uniform float belly_radius = 0.46;
uniform float belly_width  = 1.3;   // ellipse: bulge reaches wider than tall

void fragment() {
    vec2 d = UV - belly_center;
    float r = length(vec2(d.x / belly_width, d.y)) / belly_radius;
    if (r < 1.0) {
        // f < 1 near the center: sample inward, so the texture magnifies outward
        float f = 1.0 - (0.55 * fatness) * pow(1.0 - r * r, 1.5);
        COLOR = texture(TEXTURE, belly_center + d * f);
    } else {
        COLOR = texture(TEXTURE, UV);
    }
}
Applying the trait (GDScript)
# Enemy definition: all enemies are fatten-eligible unless a designer opts one out.
@export var can_fatten: bool = true

# When the Hoarder trait rolls (5% of non-boss spawns) and can_fatten is true:
sprite.material = hoarder_material.duplicate()  # per-instance copy; trivial at 5% spawn rate
sprite.material.set_shader_parameter("fatness", 0.83)

# Idle "heavy bounce": bulge stays constant, squash oscillates B -> C -> B.
# Per frame (feet-anchored sprite); abs(sin) makes the turnaround read as a body settling:
var p := absf(sin(time * TAU / 1.1))
sprite.scale = Vector2(1.0 + 0.30 * fatness * p, 1.0 - 0.12 * fatness * p)
Feasibility Notes
  • Verdict: fully feasible and cheap. The shader works in normalized UV space, so the identical material applies to every enemy sprite regardless of resolution or silhouette — exactly the “one treatment reused for all” the asset list calls for.
  • Padding requirement. The bulge pushes pixels outward, so each sprite frame needs ~15% transparent margin around the body. If frames are authored tight, the belly clips at the frame edge. This is the only real asset-side constraint.
  • Pixel-art fidelity. The game renders at 960×540 native and integer-scales, so the shader resamples at art resolution and the distortion stays on the pixel grid. Curved outlines can go slightly lumpy at high strength — keep the effective strength ≤ ~0.6 (the 0.55 factor in the shader caps this).
  • Per-enemy tuning is optional, not required. Default belly_center works for humanoids; a quadruped like the Wolf can nudge it lower. That’s one exported vec2, not new art.
  • Material sharing. Duplicate the material per hoarder instance (negligible at 5% spawns), or use instance uniform if the Godot version supports it in canvas_item shaders — then one material serves every hoarder with per-instance fatness.
  • Chosen defaults. fatness = 0.83, idle "heavy bounce" between B and C (bulge constant, squash oscillating with abs(sin) timing, period ~1.1s) — the live column above previews it. Squash is pure scale, so the animation has zero shader cost per frame.
  • Per-enemy opt-out. Enemy definitions carry can_fatten = true by default, so every enemy is fatten-eligible unless a designer disables it for a specific one. An opted-out enemy that rolls Hoarder keeps the stats and falls back to the coin-purse overlay alone.
  • Extras that come free. The coin-purse overlay from the asset list layers on top independently — the shader doesn’t conflict with it. Full written spec: LordofHirelings_HoarderFattenTech.md.
